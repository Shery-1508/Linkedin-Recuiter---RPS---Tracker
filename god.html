Firebase & scope

GOD mode talks directly to the same Firebase project as the extension. No rule changes are required; it just reads/writes extra /accounts/<accountId>/config data and manages /users and /admins.

Accounts / Teams

Each account ID represents a shared RPS account / team. Config is stored in /accounts/<accountId>/config.

Extension users & admins

These live in /users and /admins. Admins are just users with a flag in /admins. Force logout is triggered by updating a user.

If set, stored as teamId on the user record.

Inspect sessions / events (read-only)

Quick peek into /accounts/<accountId>/sessions and /accounts/<accountId>/events for troubleshooting.
